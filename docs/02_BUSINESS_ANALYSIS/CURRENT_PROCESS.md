# Current Business Process Analysis

## 1. Overview

Before the implementation of the STEG Smart Internship & Administrative Workflow Platform, internship management processes are mainly handled through manual and semi-digital procedures.

The current process relies on:

- Paper documents
- Physical signatures
- Emails exchanges
- Manual tracking through spreadsheets
- Direct communication between candidates, interns, supervisors and administrative departments

This approach creates several operational limitations related to traceability, processing time, data consistency and communication efficiency.

The objective of this analysis is to document the current situation (AS-IS process) before introducing the future digital solution (TO-BE process).

# 3. Internship Application Process

## 3.1 Candidate Request

The process starts when a student submits an internship request.

The candidate provides:

- Internship request letter
- CV
- University documents
- Identity documents
- Additional administrative files depending on requirements


Current methods:

- Physical submission
- Email transmission
- Manual document delivery


Actors involved:

| Actor | Responsibility |
|-|-|
| Candidate | Submit internship request |
| HR Department | Receive and organize requests |
| Administrative Agent | Verify documents |

---

# 4. Document Verification Process

After receiving the request, administrative agents manually verify submitted documents.

Verification includes:

- Completeness check
- Document validity
- Candidate information verification
- Internship period verification


Current problems:

- Difficult document tracking
- Missing files detection requires manual checking
- No centralized document repository

---

# 5. Application Review and Decision

After document verification:

1. Application is reviewed internally
2. Availability of internship capacity is checked
3. Decision is taken


Possible outcomes:


Application
|
+--> Accepted
|
+--> Rejected
|
+--> Pending



The decision process currently depends heavily on manual communication.

---

# 6. Internship Assignment Process

After acceptance:

The intern is assigned to:

- A department
- A supervisor
- An internship subject/project


Assignment information is managed manually.

Main involved actors:

| Actor | Responsibility |
|-|-|
| HR Manager | Manage internship allocation |
| Department Manager | Validate placement |
| Supervisor | Follow intern progress |

---

# 7. Internship Execution Process

During the internship:

The intern performs daily activities under supervisor guidance.

Current follow-up methods:

- Meetings
- Emails
- Informal communication
- Paper reports


The current process lacks:

- Digital daily journal
- Task tracking
- Deliverable management
- Online validation workflow

---

# 8. Evaluation and Completion Process

At the end of the internship:

The following activities are performed:

- Final report submission
- Supervisor evaluation
- Administrative validation
- Internship certificate preparation


Current limitations:

- Evaluation history is difficult to retrieve
- No centralized internship archive
- Manual certificate generation

# 12. Conclusion

The current internship management process successfully allows STEG to manage internship requests, but it suffers from important limitations caused by manual operations.

The absence of a centralized digital platform creates inefficiencies in:

- Application processing
- Document management
- Workflow tracking
- Communication
- Evaluation management
- Reporting

These limitations justify the development of the STEG Smart Internship & Administrative Workflow Platform.

The future solution aims to transform this manual process into a secure, automated and traceable digital ecosystem.
