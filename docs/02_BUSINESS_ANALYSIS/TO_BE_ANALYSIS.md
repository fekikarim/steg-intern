# 1. Introduction

## 1.1 Purpose

This document describes the **target future state (TO-BE)** of internship management and administrative workflows after the implementation of the **STEG Smart Internship & Administrative Workflow Platform**.

The objective is to define how current manual processes will evolve into a:

- Digitalized
- Centralized
- Automated
- Secure
- Traceable
- User-oriented

internship management ecosystem.

This document serves as a reference for:

- Software architects
- Backend developers
- Frontend developers
- Mobile developers
- Business analysts
- AI development assistants
- Future maintenance teams

to understand the expected operational behavior of the final solution.

# 2. TO-BE Vision

The future system introduces a complete digital workflow platform replacing fragmented administrative processes.

The platform provides:

- A public internship application portal.
- An internal STEG management dashboard.
- A mobile internship companion application.
- Automated document generation.
- Digital workflow validation.
- Real-time notifications.
- Centralized internship monitoring.
- Complete audit traceability.

The internship lifecycle becomes fully managed from application submission until internship completion and evaluation.

# 5. Future Process Description

---

# 5.1 Digital Internship Application

## Current Situation

Applications are mainly received through traditional methods.

## Future State

The candidate accesses the public portal and:

1. Creates an application.
2. Provides required information.
3. Uploads documents.
4. Submits the request.

The system automatically:

- Generates a unique reference.
- Stores documents securely.
- Creates an application record.
- Starts the workflow process.

---

# 5.2 Automated Application Workflow

After submission:


SUBMITTED
|
↓
UNDER_REVIEW
|
↓
ACCEPTED / REJECTED


Each transition is:

- Controlled.
- Recorded.
- Associated with a responsible user.
- Auditable.

---

# 5.3 Internship Creation

When an application is accepted:

The system creates:


Candidate
|
↓
Internship
|
↓
Assignment


The internship receives:

- Unique reference.
- Start date.
- End date.
- Department.
- Supervisor.

---

# 5.4 Supervisor Assignment

HR users assign:

- Internship department.
- Supervisor.
- Internship period.

The assigned supervisor receives:

- Notification.
- Access permissions.
- Intern information.

---

# 5.5 Digital Internship Monitoring

During internship execution:

The intern uses the mobile application to manage:

## Journal

The intern can:

- Create daily entries.
- Describe performed activities.
- Submit reports.

## Tasks

The system manages:

- Assigned tasks.
- Progress status.
- Deadlines.

## Deliverables

Interns can upload:

Supervisors can:

# 5.6 Evaluation Management

- Evaluation results.
